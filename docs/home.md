## 4-Week Learning Path

### Week 1: Foundations

- Day 1: [LLD Process](week1/day1/README) — Systematic approach to design
- Day 2: [SOLID Principles](week1/day2/README) — Core design principles
- Day 3: [UML Diagrams](week1/day3/README) — Class, Sequence, State
- Day 4: [Value Objects](week1/day4/README) — Domain modeling
- Day 5: [Error Handling](week1/day5/README) — Robust design
- Weekend: [Parking Lot](week1/weekend/README) — Full project

### Week 2: Design Patterns

- Day 6: [Creational](week2/day6/README) — Builder, Factory, Prototype
- Day 7: [Structural](week2/day7/README) — Adapter, Decorator, Proxy
- Day 8: [Behavioral](week2/day8/README) — Strategy, Observer, State
- Day 9: [Repository](week2/day9/README) — Data access patterns
- Day 10: [Caching](week2/day10/README) — LRU, LFU, TTL
- Weekend: [Elevator](week2/weekend/README) — Full project

### Week 3: System Components

- Day 11: [Rate Limiter](week3/day11/README) — Token bucket, Sliding window
- Day 12: [Notification](week3/day12/README) — Multi-channel notifications
- Day 13: [Feature Flags](week3/day13/README) — Configuration management
- Day 14: [KV Store](week3/day14/README) — WAL, Snapshots
- Day 15: [Search/Index](week3/day15/README) — Inverted index
- Weekend: [BookMyShow](week3/weekend/README) — Full project

### Week 4: Advanced

- Day 16: [Splitwise](week4/day16/README) — Debt simplification
- Day 17: [Chess/TicTacToe](week4/day17/README) — Rules engine
- Day 18: [Logging/Metrics](week4/day18/README) — MDC, async logging
- Day 19: [Review](week4/day19/README) — Code review & refactoring
- Day 20: [Mock Interviews](week4/day20/README) — Interview prep
- Weekend: [Capstone](week4/weekend/README) — Final project
