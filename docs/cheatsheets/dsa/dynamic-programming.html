Dynamic Programming // Patterns

Every DP problem follows the same 5 steps. Learn the patterns, not the problems. Recognize the pattern → write the recurrence → fill the table.

Optimal Substructure Overlapping Subproblems Memo / Tabulation 8 Core Patterns

01 — Mental Model

DP = recursion + caching. Every DP problem has these two properties:

PropertyWhat it meansTest
Optimal substructure Best answer to big problem = combination of best answers to smaller problems Can you write: solve(n) = f(solve(n-1), solve(n-2), ...)?
Overlapping subproblems Same smaller problem appears multiple times in recursion tree Draw the tree — do you see repeated nodes?

Two implementation styles — same idea:

Top-Down (Memoization)Bottom-Up (Tabulation)
DirectionStart from solve(n), recurse downStart from dp[0], build up
ImplementationRecursion + hashmap/array cacheFor-loops filling a table
ProsNatural thinking, only solves needed statesNo stack overflow, easier to optimize space
ConsStack overflow for large nMust figure out fill order

02 — The 5-Step Framework

Apply this to every DP problem:

  1. Define state — what does dp[i] (or dp[i][j]) represent?
  2. Recurrence — how does dp[i] relate to smaller states?
  3. Base case — what are the trivially known values?
  4. Fill order — which direction ensures dependencies are solved first?
  5. Answer — where in the table is the final answer?

Example: Fibonacci

StepFibonacci
1. Statedp[i] = i-th Fibonacci number
2. Recurrencedp[i] = dp[i-1] + dp[i-2]
3. Base casedp[0] = 0, dp[1] = 1
4. Fill orderLeft to right (i = 2 → n)
5. Answerdp[n]
int fib(int n) {
    if (n <= 1) return n;
    vector<int> dp(n + 1);
    dp[0] = 0; dp[1] = 1;
    for (int i = 2; i <= n; i++)
        dp[i] = dp[i - 1] + dp[i - 2];
    return dp[n];
}

03 — Pattern 1: 1D DP (Climbing Stairs Family)

State depends on a fixed number of previous states.

ProblemRecurrence
Climbing Stairs (LC 70)dp[i] = dp[i-1] + dp[i-2]
House Robber (LC 198)dp[i] = max(dp[i-1], dp[i-2] + nums[i])
Min Cost Climbing (LC 746)dp[i] = cost[i] + min(dp[i-1], dp[i-2])
Decode Ways (LC 91)dp[i] = dp[i-1] (if valid 1-digit) + dp[i-2] (if valid 2-digit)

House Robber (LC 198)

// dp[i] = max money robbing houses 0..i
// Choice: skip house i OR rob it (can't rob i-1)
int rob(vector<int>& nums) {
    int n = nums.size();
    if (n == 1) return nums[0];
    int prev2 = nums[0];
    int prev1 = max(nums[0], nums[1]);
    for (int i = 2; i < n; i++) {
        int curr = max(prev1, prev2 + nums[i]);
        prev2 = prev1;
        prev1 = curr;
    }
    return prev1;
}

Space optimized to O(1) since we only need 2 previous values.

04 — Pattern 2: 0/1 Knapsack

Template: Given items with weight & value, each used at most once, maximize value within capacity.

StepValue
Statedp[i][w] = max value using items 0..i with capacity w
Recurrencedp[i][w] = max(dp[i-1][w], dp[i-1][w - wt[i]] + val[i])
ChoiceSkip item i OR take it (if it fits)
Basedp[0][w] = 0 for all w
Answerdp[n][W]
// Classic 0/1 Knapsack
int knapsack(vector<int>& wt, vector<int>& val, int W) {
    int n = wt.size();
    vector<int> dp(W + 1, 0);
    for (int i = 0; i < n; i++)
        for (int w = W; w >= wt[i]; w--)     // RIGHT to LEFT (each item once)
            dp[w] = max(dp[w], dp[w - wt[i]] + val[i]);
    return dp[W];
}
Key: iterate w from right to left so each item is only used once. Left to right would reuse items (unbounded knapsack).

Partition Equal Subset Sum (LC 416)

// Can we pick a subset summing to sum/2?
bool canPartition(vector<int>& nums) {
    int sum = 0;
    for (int x : nums) sum += x;
    if (sum % 2) return false;
    int target = sum / 2;

    vector<bool> dp(target + 1, false);
    dp[0] = true;
    for (int num : nums)
        for (int j = target; j >= num; j--)  // right to left
            dp[j] = dp[j] || dp[j - num];
    return dp[target];
}

05 — Pattern 3: Unbounded Knapsack

Same as 0/1, but each item can be used unlimited times. Only difference: iterate w from left to right.

0/1 KnapsackUnbounded Knapsack
Each item usedAt most onceUnlimited times
Inner loop directionw = W → wt[i] (right to left)w = wt[i] → W (left to right)
WhyDon't read already-updated valuesDO read updated values (reuse allowed)

Coin Change (LC 322) — Min coins to make amount

int coinChange(vector<int>& coins, int amount) {
    vector<int> dp(amount + 1, amount + 1);  // init to "impossible"
    dp[0] = 0;
    for (int coin : coins)
        for (int j = coin; j <= amount; j++)  // LEFT to RIGHT (reuse)
            dp[j] = min(dp[j], dp[j - coin] + 1);
    return dp[amount] > amount ? -1 : dp[amount];
}

Coin Change II (LC 518) — Count combinations

// Order doesn't matter: [1,2] and [2,1] are SAME combination
int change(int amount, vector<int>& coins) {
    vector<int> dp(amount + 1, 0);
    dp[0] = 1;
    for (int coin : coins)                  // outer: coins
        for (int j = coin; j <= amount; j++)  // inner: amount
            dp[j] += dp[j - coin];
    return dp[amount];
}
Combinations vs Permutations: Coins in outer loop → combinations (each coin considered once). Amount in outer loop → permutations (LC 377 Combination Sum IV).
ProblemLoop OrderResult
Coin Change II (LC 518)Coins outer, amount innerCombinations (order-independent)
Combination Sum IV (LC 377)Amount outer, coins innerPermutations (order matters)

06 — Pattern 4: LCS / Two-String DP

Template: Compare two sequences character by character. State: dp[i][j] = answer for first i chars of s1 and first j chars of s2.

Longest Common Subsequence (LC 1143)

int longestCommonSubsequence(string& s1, string& s2) {
    int m = s1.size(), n = s2.size();
    vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0));

    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++)
            if (s1[i-1] == s2[j-1])
                dp[i][j] = dp[i-1][j-1] + 1;  // match: extend diagonal
            else
                dp[i][j] = max(dp[i-1][j], dp[i][j-1]);  // skip one char

    return dp[m][n];
}

Problems built on LCS

ProblemReduction
Edit Distance (LC 72)3 operations: dp[i][j] = min(insert, delete, replace)

Shortest Common Supersequence (LC 1092)m + n - LCS
Min Deletions to Make Equal (LC 583)m + n - 2 * LCS
Longest Palindromic Subsequence (LC 516)LCS(s, reverse(s))

Edit Distance (LC 72)

int minDistance(string& s1, string& s2) {
    int m = s1.size(), n = s2.size();
    vector<vector<int>> dp(m + 1, vector<int>(n + 1, 0));

    for (int i = 0; i <= m; i++) dp[i][0] = i;  // delete all
    for (int j = 0; j <= n; j++) dp[0][j] = j;  // insert all

    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++)
            if (s1[i-1] == s2[j-1])
                dp[i][j] = dp[i-1][j-1];           // match: no cost
            else
                dp[i][j] = 1 + min({dp[i-1][j],     // delete
                                   dp[i][j-1],     // insert
                                   dp[i-1][j-1]});  // replace
    return dp[m][n];
}

07 — Pattern 5: Longest Increasing Subsequence

Two approaches: O(n²) DP and O(n log n) with binary search.

O(n²) DP

// dp[i] = length of LIS ending at index i
int lengthOfLIS(vector<int>& nums) {
    int n = nums.size();
    vector<int> dp(n, 1);
    int ans = 1;

    for (int i = 1; i < n; i++) {
        for (int j = 0; j < i; j++)
            if (nums[j] < nums[i])
                dp[i] = max(dp[i], dp[j] + 1);
        ans = max(ans, dp[i]);
    }
    return ans;
}

O(n log n) with Binary Search (Patience Sorting)

int lengthOfLIS(vector<int>& nums) {
    vector<int> tails;  // tails[i] = smallest tail of LIS of length i+1

    for (int x : nums) {
        auto it = lower_bound(tails.begin(), tails.end(), x);
        if (it == tails.end())
            tails.push_back(x);    // extend longest
        else
            *it = x;                // replace to keep tails small
    }
    return tails.size();
}

tails is always sorted. lower_bound finds where to place x. The array length = LIS length. (Note: the array itself is NOT the LIS.)

08 — Pattern 6: Grid DP

State: dp[i][j] = answer at cell (i, j). Fill top-left to bottom-right.

Unique Paths (LC 62)

int uniquePaths(int m, int n) {
    vector<vector<int>> dp(m, vector<int>(n, 1));
    for (int i = 1; i < m; i++)
        for (int j = 1; j < n; j++)
            dp[i][j] = dp[i-1][j] + dp[i][j-1];  // from top + from left
    return dp[m-1][n-1];
}

Minimum Path Sum (LC 64)

int minPathSum(vector<vector<int>>& grid) {
    int m = grid.size(), n = grid[0].size();
    for (int i = 0; i < m; i++)
        for (int j = 0; j < n; j++) {
            if (i == 0 && j == 0) continue;
            else if (i == 0) grid[i][j] += grid[i][j-1];
            else if (j == 0) grid[i][j] += grid[i-1][j];
            else grid[i][j] += min(grid[i-1][j], grid[i][j-1]);
        }
    return grid[m-1][n-1];
}

09 — Pattern 7: Interval DP

State: dp[i][j] = answer for the subarray/substring from i to j. Try every split point k between i and j.

// Generic interval DP template
for (int len = 2; len <= n; len++)          // enumerate length
    for (int i = 0; i + len - 1 < n; i++) {
        int j = i + len - 1;                  // right endpoint
        for (int k = i; k < j; k++)            // try every split
            dp[i][j] = min(dp[i][j], dp[i][k] + dp[k+1][j] + cost(i, j));
    }

Burst Balloons (LC 312)

// dp[i][j] = max coins bursting all balloons between i and j
int maxCoins(vector<int>& nums) {
    int n = nums.size();
    nums.insert(nums.begin(), 1);
    nums.push_back(1);  // pad with 1s
    vector<vector<int>> dp(n + 2, vector<int>(n + 2, 0));

    for (int len = 1; len <= n; len++)
        for (int i = 1; i + len - 1 <= n; i++) {
            int j = i + len - 1;
            for (int k = i; k <= j; k++)  // k = last balloon to burst
                dp[i][j] = max(dp[i][j],
                    dp[i][k-1] + nums[i-1]*nums[k]*nums[j+1] + dp[k+1][j]);
        }
    return dp[1][n];
}

Time: O(n³). Space: O(n²). Also applies to: Matrix Chain Multiplication, Palindrome Partitioning II, Stone Game.

10 — Pattern 8: Bitmask DP

When the state involves a subset of items (n ≤ 20), represent the subset as a bitmask.

dp[mask] = answer when the items in mask have been visited/used.

Travelling Salesman (classic)

// dp[mask][i] = min cost to visit all cities in mask, ending at city i
int n = dist.size();
vector<vector<int>> dp(1 << n, vector<int>(n, INT_MAX));
dp[1][0] = 0;  // start at city 0

for (int mask = 1; mask < (1 << n); mask++)
    for (int u = 0; u < n; u++) {
        if (!(mask & (1 << u)) || dp[mask][u] == INT_MAX) continue;
        for (int v = 0; v < n; v++) {
            if (mask & (1 << v)) continue;  // already visited
            int next = mask | (1 << v);
            dp[next][v] = min(dp[next][v], dp[mask][u] + dist[u][v]);
        }
    }

Time: O(2n · n²). Space: O(2n · n). Only feasible for n ≤ 20.

Bit OperationMeaning
mask & (1 << i)Is item i in the subset?
mask | (1 << i)Add item i to subset
mask ^ (1 << i)Toggle item i
mask & (mask - 1)Remove lowest set bit
__builtin_popcount(mask)Count items in subset

If dp[i] only depends on dp[i-1] (previous row), you can reduce 2D → 1D or use two variables.

0/1 Knapsack: 2D → 1D

// 2D version: dp[i][w] = max value with items 0..i, capacity w
// dp[i][w] only reads from dp[i-1][...] → collapse to 1D
// Iterate w RIGHT to LEFT so we don't overwrite values we still need

vector<int> dp(W + 1, 0);
for (int i = 0; i < n; i++)
    for (int w = W; w >= wt[i]; w--)     // RIGHT → LEFT

12 — Top-Down vs Bottom-Up

Top-Down (Memo)Bottom-Up (Tab)
Code styleRecursiveIterative
Start fromFinal answer, recurse to baseBase case, build to answer
States computedOnly reachable statesAll states in the table
Stack overflow?Yes, for large nNo
Space optimize?HarderEasier (rolling array)
When to useComplex state transitions, tree DPStandard patterns, interviews

Same Problem, Both Ways

// TOP-DOWN: Coin Change with memoization
unordered_map<int, int> memo;
int solve(vector<int>& coins, int amount) {
    if (amount == 0) return 0;
    if (amount < 0) return 1e9;
    if (memo.count(amount)) return memo[amount];

    int res = 1e9;
    for (int c : coins)
        res = min(res, 1 + solve(coins, amount - c));
    return memo[amount] = res;
}
// BOTTOM-UP: Coin Change with tabulation
int coinChange(vector<int>& coins, int amount) {
    vector<int> dp(amount + 1, amount + 1);
    dp[0] = 0;
    for (int c : coins)
        for (int j = c; j <= amount; j++)
            dp[j] = min(dp[j], dp[j - c] + 1);
    return dp[amount] > amount ? -1 : dp[amount];
}

14 — Key Reminders

  • Start with brute-force recursion, add memoization, then convert to tabulation
  • If you can't define the state clearly, you can't solve the DP
  • 0/1 Knapsack: inner loop goes right to left (prevent reuse)
  • Unbounded Knapsack: inner loop goes left to right (allow reuse)
  • Combinations: coins in outer loop. Permutations: amount in outer loop
  • For string DP, use 1-indexed table (dp size = m+1 × n+1) to handle empty string base case
  • Interval DP: always enumerate by length first, then start index
  • If DP state depends on only the previous row/state, optimize space
  • "Minimize the maximum" or "maximize the minimum" is usually binary search, not DP
  • If n ≤ 20 and you need subsets → bitmask DP
