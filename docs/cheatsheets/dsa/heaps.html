01 — PriorityQueue API Quick Ref

MethodWhat it doesReturnsModifies heap?
offer(e) Insert element into heap true if added Yes — adds + re-heapifies
peek() Look at head, don't remove Head element or null No — read-only
poll() Remove and return head Head element or null Yes — removes + re-heapifies

offer/poll/peek return null on failure. add/remove/element throw exceptions. Prefer the null-safe versions.

// Min-heap (default)
PriorityQueue<Integer> minHeap = new PriorityQueue<>();

// Max-heap
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());

02 — Find Median from Data Stream (LC 295)

Core Idea

Split the stream into two halves using two heaps. The tops of both heaps are always the two middle elements.

HeapTypeStoresPeek returns
maxHeap (left) Max-Heap Smaller half Largest of small half
minHeap (right) Min-Heap Larger half Smallest of large half 
[1, 3, 5, | 7, 9, 11]
 ^^^^^^^^    ^^^^^^^^^
 maxHeap     minHeap
 peek→5     peek→7
 median = (5 + 7) / 2 = 6.0

Algorithm (3 Steps)

  1. Always push to maxHeap first
  2. Move maxHeap's top to minHeap (ensures left ≤ right)
  3. If minHeap grows larger, move its top back to maxHeap (keeps sizes balanced)
Without step 3, every element drains into minHeap. Step 1 puts it in maxHeap, step 2 immediately moves it to minHeap. Step 3 is the only thing that pulls elements back to the left half.

Java Code

class MedianFinder {
    private PriorityQueue<Integer> maxHeap; // left half
    private PriorityQueue<Integer> minHeap; // right half

    public MedianFinder() {
        maxHeap = new PriorityQueue<>(Collections.reverseOrder());
        minHeap = new PriorityQueue<>();
    }

    public void addNum(int num) {
        maxHeap.offer(num);
        minHeap.offer(maxHeap.poll());
        if (minHeap.size() > maxHeap.size())
            maxHeap.offer(minHeap.poll());
    }

    public double findMedian() {
        if (maxHeap.size() > minHeap.size())
            return maxHeap.peek();
        return (maxHeap.peek() + minHeap.peek()) / 2.0;
    }
}

addNumO(log n)O(n)

03 — Meeting Rooms II (LC 253)

Core Idea

Find the minimum number of conference rooms needed. Equivalent to finding the maximum overlap of intervals.

Approach 1: Sort + Min-Heap

Sort by start. Min-heap tracks the earliest ending meeting. If a new meeting starts after earliest end, reuse that room.

int minMeetingRooms(vector<vector<int>>& intervals) {
    sort(intervals.begin(), intervals.end());
    priority_queue<int, vector<int>, greater<>> heap;
    for (auto& m : intervals) {
        if (!heap.empty() && heap.top() <= m[0])
            heap.pop();
        heap.push(m[1]);
    }
    return heap.size();
}

Approach 2: Two Pointers on Sorted Start/End

Separate start and end arrays, sort each. Every start = +1 room, every end = -1 room.

int minMeetingRooms(vector<vector<int>>& intervals) {
    int n = intervals.size();
    vector<int> starts(n), ends(n);
    for (int i = 0; i < n; i++) {
        starts[i] = intervals[i][0];
        ends[i] = intervals[i][1];
    }
    sort(starts.begin(), starts.end());
    sort(ends.begin(), ends.end());

    int rooms = 0, maxRooms = 0, s = 0, e = 0;
    while (s < n) {
        if (starts[s] < ends[e]) { rooms++; s++; }
        else { rooms--; e++; }
        maxRooms = max(maxRooms, rooms);
    }
    return maxRooms;
}

Approach 3: Sweep Line (Event-Based)

+1 at start, -1 at end. Sort events. Running sum = active rooms. Max = answer.

int minMeetingRooms(vector<vector<int>>& intervals) {
    map<int, int> events;
    for (auto& m : intervals) {
        events[m[0]]++;
        events[m[1]]--;
    }
    int rooms = 0, maxRooms = 0;
    for (auto& [time, delta] : events) {
        rooms += delta;
        maxRooms = max(maxRooms, rooms);
    }
    return maxRooms;
}

Complexity

ApproachTimeSpaceStrength
Min-HeapO(n log n)O(n)Intuitive; simulates real room allocation
Two PointersO(n log n)O(n)No extra data structure; slickest
Sweep LineO(n log n)O(n)Generalizes to any overlap problem

04 — Evaluate Division (LC 399)

Core Idea

Model equations as a directed weighted graph. a/b = 2.0 means edge a→b weight 2.0 and b→a weight 0.5. Path cost = product of edges.

a —(2.0)→ b —(3.0)→ c
a / c = (a/b) × (b/c) = 2.0 × 3.0 = 6.0

Approach 1: BFS per Query (Interview Default)

double bfs(map<string, map<string, double>>& graph,
           string src, string dst) {
    if (!graph.count(src) || !graph.count(dst)) return -1.0;
    if (src == dst) return 1.0;

    queue<pair<string, double>> q;
    set<string> visited;
    q.push({src, 1.0});
    visited.insert(src);

    while (!q.empty()) {
        auto [node, product] = q.front(); q.pop();
        for (auto& [neighbor, weight] : graph[node]) {
            if (neighbor == dst) return product * weight;
            if (!visited.count(neighbor)) {
                visited.insert(neighbor);
                q.push({neighbor, product * weight});
            }
        }
    }
    return -1.0;
}

Approach 2: Floyd-Warshall (All Pairs)

Precompute every pair. Relaxation: dist[i][j] = dist[i][k] * dist[k][j]

O(V³) precompute, O(1) per query. Best for many queries, few variables.

Approach 3: Union-Find with Weights

Each node stores ratio to root. Query: ratio[a] / ratio[b] = (a/root) / (b/root) = a/b

O(α(n)) per query ≈ O(1). Best for dynamic edges + many queries.
