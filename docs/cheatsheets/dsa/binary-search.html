Binary Search // Templates

Three templates that cover every binary search problem. Define search space, define condition, pick the template.

O(log n) 3 Templates 90% = First True

01 — Mental Model

Every binary search is just:

  1. Define your search space: [lo, hi]
  2. Define your condition: condition(mid) → true/false
  3. Pick the template:
WantActionTemplate
FIRST true Save ans, go LEFT hi = mid - 1 Template 2
LAST true Save ans, go RIGHT lo = mid + 1 Template 3

02 — Template 1: Exact Match

Find exact target in sorted array. Return index or -1.

int binarySearch(vector<int>& nums, int target) {
    int lo = 0, hi = nums.size() - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (nums[mid] == target) return mid;
        else if (nums[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}

Use when: simple search, exact value lookup.

03 — Template 2: First True (Most Important)

Array looks like: [false, false, ..., true, true, true] → find the first true

int firstTrue(int lo, int hi, auto condition) {
    int ans = -1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (condition(mid)) {
            ans = mid;        // could be the answer
            hi = mid - 1;     // but look left for earlier one
        } else {
            lo = mid + 1;     // too early, go right
        }
    }
    return ans;
}
This covers 90% of binary search problems. Just change the condition.

04 — Template 3: Last True

Array looks like: [true, true, true, ..., false, false] → find the last true

int lastTrue(int lo, int hi, auto condition) {
    int ans = -1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2;
        if (condition(mid)) {
            ans = mid;        // could be the answer
            lo = mid + 1;     // but look right for later one
        } else {
            hi = mid - 1;     // too far, go left
        }
    }
    return ans;
}

05 — Problem Mapping

ProblemTemplateCondition
Find target Exact nums[mid] == target
First occurrence of target First true nums[mid] >= target
Last occurrence of target Last true nums[mid] <= target
Lower bound (first >= x) First true nums[mid] >= x
Upper bound (first > x) First true nums[mid] > x
Search in rotated array Exact + check Which half is sorted?
Min in rotated array First true nums[mid] <= nums[hi]
Koko eating bananas First true canFinish(mid, piles, h)
Ship packages in D days First true canShip(mid, weights, days)
Sqrt(x) Last true mid * mid <= x
Peak element Boundary nums[mid] > nums[mid+1]

06 — Key Reminders

  • Always use mid = lo + (hi - lo) / 2 to avoid integer overflow
  • Loop condition is always while (lo <= hi)
  • Never do lo = mid or hi = mid (infinite loop risk) — always +1 or -1
  • 90% of problems are Template 2 (first true) with a custom condition
  • For "minimize the maximum" or "maximize the minimum" → binary search on the answer + feasibility check
