Backtracking // Templates

One skeleton that solves subsets, permutations, combinations, N-Queens, Sudoku, and more. Learn the pattern, swap the constraint.

Recursion + Undo 3 Core Patterns Pruning = Speed O(2^n) / O(n!)

01 — Mental Model

Backtracking is DFS on a decision tree. At each node you:

  1. Choose — add a candidate to the current path
  2. Explore — recurse deeper
  3. Un-choose — remove it (backtrack) and try the next candidate

The key insight: prune early. If the current path can never lead to a valid solution, stop exploring it immediately.

[] / | \ [1] [2] [3] / \ | [1,2] [1,3] [2,3] ← explore | [1,2,3] ← valid? collect if invalid → PRUNE (backtrack)

02 — Generic Template (The Skeleton)

This is the one template you adapt for every backtracking problem:

void backtrack(vector<vector<int>>& result,
              vector<int>& path,
              /* input + state */) {

    // Base case: found a valid solution
    if (isSolution(path)) {
        result.push_back(path);
        return;
    }

    for (auto& candidate : getCandidates(state)) {
        if (!isValid(candidate)) continue;  // prune

        path.push_back(candidate);    // choose
        backtrack(result, path, ...);  // explore
        path.pop_back();               // un-choose
    }
}
Every backtracking problem is just this skeleton with different: (1) candidates, (2) validity check, (3) base case.
PieceWhat changes per problem
isSolution() When to collect: path.size() == n? All cells filled? Sum == target?
getCandidates() What to try next: remaining numbers? digits 1-9? directions? letters?
isValid() Pruning: already used? column conflict? sum exceeded? duplicate?

03 — Pattern 1: Subsets (Include / Skip)

At each index, two choices: include it or skip it. Collect at every node.

// LC 78: Subsets
vector<vector<int>> subsets(vector<int>& nums) {
    vector<vector<int>> result;
    vector<int> path;
    backtrack(nums, 0, path, result);
    return result;
}

void backtrack(vector<int>& nums, int start,
              vector<int>& path, vector<vector<int>>& result) {
    result.push_back(path);         // collect every node

    for (int i = start; i < nums.size(); i++) {
        path.push_back(nums[i]);     // choose
        backtrack(nums, i + 1, path, result);  // explore (i+1 = no reuse)
        path.pop_back();              // un-choose
    }
}
Key detailWhy
start = i + 1Each element used at most once, no backwards
Collect at every nodeEvery subset (including empty) is valid
No explicit base caseLoop naturally ends when start == n

04 — Pattern 2: Permutations (Use Each Once)

Try every element at every position, but skip already-used ones. Collect only when path is full length.

// LC 46: Permutations
vector<vector<int>> permute(vector<int>& nums) {
    vector<vector<int>> result;
    vector<int> path;
    vector<bool> used(nums.size(), false);
    backtrack(nums, path, used, result);
    return result;
}

void backtrack(vector<int>& nums, vector<int>& path,
              vector<bool>& used, vector<vector<int>>& result) {
    if (path.size() == nums.size()) {
        result.push_back(path);      // collect only full permutations
        return;
    }

    for (int i = 0; i < nums.size(); i++) {
        if (used[i]) continue;       // skip already used

        used[i] = true;              // choose
        path.push_back(nums[i]);
        backtrack(nums, path, used, result);  // explore
        path.pop_back();              // un-choose
        used[i] = false;
    }
}

05 — Pattern 3: Combinations (Pick k from n)

Same as subsets, but collect only when path.size() == k. Add an early prune if not enough elements remaining.

// LC 77: Combinations
vector<vector<int>> combine(int n, int k) {
    vector<vector<int>> result;
    vector<int> path;
    backtrack(n, k, 1, path, result);
    return result;
}

void backtrack(int n, int k, int start,
              vector<int>& path, vector<vector<int>>& result) {
    if (path.size() == k) {
        result.push_back(path);
        return;
    }

    // Prune: need (k - path.size()) more, only (n - i + 1) left
    for (int i = start; i <= n - (k - path.size()) + 1; i++) {
        path.push_back(i);
        backtrack(n, k, i + 1, path, result);
        path.pop_back();
    }
}

Combination Sum (reuse allowed)

Same structure, but pass i instead of i + 1 to allow reuse. Prune when sum exceeds target.

// LC 39: Combination Sum
void backtrack(vector<int>& cands, int target, int start,
              vector<int>& path, vector<vector<int>>& result) {
    if (target == 0) {
        result.push_back(path);
        return;
    }

    for (int i = start; i < cands.size(); i++) {
        if (cands[i] > target) break;  // prune (array is sorted)

        path.push_back(cands[i]);
        backtrack(cands, target - cands[i], i, path, result);  // i, not i+1
        path.pop_back();
    }
}
Sort candidates first so the break prune works. This is a massive speedup.

06 — Handling Duplicates

When input has duplicates (e.g. [1, 2, 2, 3]), sort first, then skip consecutive equal values at the same decision level:

// The one-line fix that eliminates duplicate results:
if (i > start && nums[i] == nums[i - 1]) continue;

Subsets II (LC 90)

void backtrack(vector<int>& nums, int start,
              vector<int>& path, vector<vector<int>>& result) {
    result.push_back(path);

    for (int i = start; i < nums.size(); i++) {
        if (i > start && nums[i] == nums[i - 1]) continue;  // skip dup

        path.push_back(nums[i]);
        backtrack(nums, i + 1, path, result);
        path.pop_back();
    }
}

Permutations II (LC 47)

void backtrack(vector<int>& nums, vector<int>& path,
              vector<bool>& used, vector<vector<int>>& result) {
    if (path.size() == nums.size()) {
        result.push_back(path);
        return;
    }

    for (int i = 0; i < nums.size(); i++) {
        if (used[i]) continue;
        // skip dup: same value, previous not used = duplicate branch
        if (i > 0 && nums[i] == nums[i - 1] && !used[i - 1]) continue;

        used[i] = true;
        path.push_back(nums[i]);
        backtrack(nums, path, used, result);
        path.pop_back();
        used[i] = false;
    }
}
Duplicate rule for permutations: skip nums[i] if it equals nums[i-1] AND nums[i-1] was NOT used. This ensures only one ordering of equal elements.
PatternDuplicate Skip Condition
Subsets / Combinationsi > start && nums[i] == nums[i-1]
Permutationsi > 0 && nums[i] == nums[i-1] && !used[i-1]

07 — Constraint Satisfaction (Board Problems)

N-Queens (LC 51)

Place N queens on an N×N board so no two attack each other. Place one queen per row, check column + diagonals.

void solve(int row, int n,
          vector<bool>& cols,
          vector<bool>& diag1,    // row - col + n - 1
          vector<bool>& diag2,    // row + col
          vector<string>& board,
          vector<vector<string>>& result) {
    if (row == n) {
        result.push_back(board);
        return;
    }

    for (int col = 0; col < n; col++) {
        if (cols[col] || diag1[row - col + n - 1] || diag2[row + col])
            continue;  // prune: conflict

        board[row][col] = 'Q';
        cols[col] = diag1[row - col + n - 1] = diag2[row + col] = true;

        solve(row + 1, n, cols, diag1, diag2, board, result);

        board[row][col] = '.';   // un-choose
        cols[col] = diag1[row - col + n - 1] = diag2[row + col] = false;
    }
}

Trick: encode diagonals as row - col and row + col. All cells on the same diagonal share the same value.

Sudoku Solver (LC 37)

bool solve(vector<vector<char>>& board) {
    for (int r = 0; r < 9; r++) {
        for (int c = 0; c < 9; c++) {
            if (board[r][c] != '.') continue;

            for (char d = '1'; d <= '9'; d++) {
                if (isValid(board, r, c, d)) {
                    board[r][c] = d;
                    if (solve(board)) return true;
                    board[r][c] = '.';  // un-choose
                }
            }
            return false;  // no digit works → backtrack
        }
    }
    return true;  // all cells filled
}

bool isValid(vector<vector<char>>& board, int r, int c, char d) {
    for (int i = 0; i < 9; i++) {
        if (board[r][i] == d) return false;          // row check
        if (board[i][c] == d) return false;          // col check
        if (board[r/3*3 + i/3][c/3*3 + i%3] == d)  // box check
            return false;
    }
    return true;
}

Key: after trying all digits 1-9 for an empty cell and none work, return false to trigger backtracking in the caller.

08 — String Partitioning

Partition a string into parts that satisfy a condition. Classic: Palindrome Partitioning (LC 131).

void backtrack(string& s, int start,
              vector<string>& path,
              vector<vector<string>>& result) {
    if (start == s.size()) {
        result.push_back(path);
        return;
    }

    for (int end = start; end < s.size(); end++) {
        if (isPalindrome(s, start, end)) {
            path.push_back(s.substr(start, end - start + 1));
            backtrack(s, end + 1, path, result);
            path.pop_back();
        }
    }
}

Same pattern applies to: Word Break II (LC 140), Restore IP Addresses (LC 93), Letter Combinations of Phone Number (LC 17).

10 — Complexity Cheat Sheet

Problem TypeTimeSpaceWhy
Subsets O(2n · n) O(n) 2n subsets, each up to n elements to copy
Permutations O(n! · n) O(n) n! permutations, each n elements to copy
Combinations C(n,k) O(C(n,k) · k) O(k) C(n,k) results, each k elements
Combination Sum O(2t/min) O(t/min) t = target, min = smallest candidate
N-Queens O(n!) O(n) n choices for row 0, n-1 for row 1, etc.
Sudoku O(9empty) O(1) 9 choices per empty cell, heavy pruning in practice
Palindrome Partition O(n · 2n) O(n) 2n possible cuts, n to check palindrome
Space above is recursion stack depth (excluding output storage). Output space is always proportional to the number of results.

11 — Key Reminders

  • Sort the input if you need to handle duplicates or enable pruning
  • Choose → Explore → Un-choose — always undo your mutation after recursion
  • start = i + 1 means “no reuse” — start = i means “reuse allowed”
  • For permutations, loop from 0 with a used[] array. For subsets/combos, loop from start
  • Prune aggressively: the earlier you reject a branch, the faster the algorithm
  • For constraint problems (N-Queens, Sudoku), the “isValid” check IS the pruning
  • Always push a copy of path to results (in C++ push_back(path) copies, in Java/Python you must explicitly copy)
  • If the problem says “return all possible...” — think backtracking
  • If the problem says “return count of...” — think DP (unless n is small)
