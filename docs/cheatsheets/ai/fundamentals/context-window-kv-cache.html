Context Window & KV Cache // Fundamentals

Context window is the attention budget; KV cache is how inference makes autoregressive decoding affordable. Understand prefill/decode, memory growth, PagedAttention, and long-context mitigations.

KV Cache Prefill PagedAttention Long Context

Each generated token appends to per-layer K and V tensors; PagedAttention allocates GPU blocks efficiently across batched requests.

01 — What & Why

Without KV cache, each new token would recompute attention over the entire history at O(n²) per step. Cache stores prior keys/values so decode is O(n) per step in sequence length. Context window caps n.

03 — Architecture

Prefill: tokens[1..n] ──► fill KV[1..n] per layer
Decode: token[n+1] ──► attend KV[1..n] ──► append KV[n+1] ──► repeat

04 — KV Cache Mechanics

For each layer, cache tensors shape roughly (batch, heads, seq_len, head_dim). New token only computes Q,K,V for itself; attends all cached K,V.

05 — Prefill Phase

Parallel over prompt positions; compute-intensive; dominates TTFT for RAG with large contexts.

06 — Decode Phase

Serial steps; memory-bandwidth bound; TPS improves with batching multiple sequences.

07 — Memory Growth

KV memory ≈ 2 * num_layers * seq_len * hidden * bytes_per_param. 128K context on 70B can exceed single GPU without offloading.

08 — PagedAttention & Long Context

Allocates KV in non-contiguous blocks like OS virtual memory; vLLM achieves higher batch utilization.

  • RoPE scaling (NTK, YaRN) for extended positions
  • Sliding window attention (Mistral) bounds KV size
  • Prefer retrieval over stuffing entire corpus

Per-layer K and V for all prior positions.

Non-contiguous GPU blocks per sequence.

12 — Interview Q&A

What is stored in KV cache?
Per transformer layer: key and value projections for each prior token position, so new queries attend without recomputation.
Prefill vs decode?
Prefill processes all prompt tokens at once (parallel). Decode generates one token at a time (serial), appending to cache each step.
Why does KV memory matter?
Serves many long conversations concurrently; OOM kills replicas; drives $/GPU-hour economics.
What is PagedAttention?
vLLM block-based KV allocation reducing fragmentation when batching variable-length sequences.
How does context window relate to KV?
Window is max seq_len n; KV size grows linearly with n per layer until cap hit.
Why is long RAG context expensive?
Large prefill + large KV for entire session; hurts TTFT, memory, and input token bill.
Multi-turn chat optimization?
Reuse cached prefix if platform supports session KV; else summarize old turns.
Sliding window attention?
Each token only attends last W positions; bounds KV memory; used in Mistral variants.
RoPE scaling for longer context?
Interpolation/extrapolation tricks extend positions beyond training length with some quality loss.
When to retrieve vs stuff context?
Retrieve top-k when corpus huge; stuffing only when total tokens affordable.
Speculative decoding relation?
Draft model proposes tokens; target verifies—speeds decode, separate from KV but same serving stack.
SDE3 KV cache answer?
Explains prefill/decode, KV growth formula intuition, PagedAttention, ties to cost/latency and RAG prompt sizing.
