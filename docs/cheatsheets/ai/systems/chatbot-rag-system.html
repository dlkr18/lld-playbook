Full system design: enterprise chatbot with RAG, conversation memory, semantic cache, agent tools, and HITL. The canonical 45-minute AI system design interview.

System Design10K QPSMulti-tenantStreamingHITL

Chatbot RAG system HLD: client apps → API gateway → chat service (session, memory, cache) → RAG pipeline → optional agent tools → LLM fleet with streaming SSE responses.

01 — What & Why

Functional Requirements

  • Multi-turn chat with conversation history
  • Answer from private enterprise knowledge base with citations
  • Execute actions (create ticket, search CRM) via agent tools
  • Stream responses token-by-token
  • Admin: ingest docs, view analytics, configure bot persona

Non-Functional Requirements

  • 10K QPS peak; p99 first token <500ms
  • Multi-tenant isolation (org_id, ACL)
  • 99.9% availability
  • GDPR: delete user data within 30 days
  • Cost budget: <$0.05 per query average

03 — Architecture

  -> Query Router -> [Cache hit] | [RAG path] | [Agent path]
  RAG: embed -> hybrid retrieve -> rerank -> LLM stream
  Agent: tool loop with HITL on writes
  Session store (Redis) + Memory (vector) + Postgres metadata

Detailed chat service components: router, session store, RAG retriever, LLM gateway, streaming SSE

03a — Estimation (1M docs, 10K QPS)

Corpus: 1M docs -> ~10M chunks
Storage: ~120GB vectors + 30GB sparse + 5GB metadata
10K QPS: semantic cache 40% hit -> 6K to LLM
LLM fleet: vLLM ~200 GPUs or tiered model routing
RAG retrieve: 50 ANN shards, rerank bottleneck -> top-20
Cost: cache saves ~40% LLM spend; avg $0.03/query achievable

05 — Deep Dive: Chat Path

  1. Auth + rate limit
  2. Load session history; summarize if >8K tokens
  3. Semantic cache lookup
  4. Router: FAQ / RAG / agent
  5. Stream LLM response via SSE
  6. Persist message + citations; async feedback

06 — Deep Dive: RAG Integration

See RAG End-to-End. Hybrid retrieve top-50, rerank top-5, inject numbered passages, generate with [1][2] citations. Refuse if max rerank score below threshold.

07 — Deep Dive: Session Memory

Redis session buffer. Rolling summary over 8K tokens. Long-term vector memory for user prefs (Agent Memory).

08 — Deep Dive: Agent Tools

Optional agent path for ticket creation, CRM lookup. HITL on writes (Production Agent). Max 8 tool calls.

09 — Production & Ops

  • Eval harness CI gate on every deploy
  • Tracing: cache/RAG/agent/LLM spans

11 — Common Pitfalls

  • No query router — every query hits expensive RAG
  • Session history unbounded
  • Missing citation verification
  • No tenant isolation on cache and retrieval

12 — Interview Q&A

Design a chatbot with RAG from scratch.
Clarify: corpus size, QPS, multi-tenant, actions needed. Draw: API GW → Chat Service → Router → Cache/RAG/Agent paths. Offline ingest pipeline. Session in Redis. Stream via SSE. Citations required.
Core entities?
Organization, User, Session, Message, Document, Chunk, Citation, ToolRun, Feedback. Session 1:N Messages. Document 1:N Chunks.
API design?
POST sessions, POST messages with stream:true → SSE, async ingest POST /documents. Idempotent message IDs for retry.
Size for 1M docs, 10K QPS?
10M chunks, 120GB vectors. Cache 40% hits. 6K QPS LLM — tiered models + vLLM fleet. Rerank bottleneck — reduce to top-20.
Router purpose?
Classify simple FAQ (cached/template), RAG-needed (knowledge), agent-needed (action). Saves cost and latency.
Multi-tenant isolation?
org_id on every query, cache namespace, vector pre-filter. Separate encryption keys per enterprise tier.
Streaming implementation?
SSE from LLM token stream. Send citations in final event. Client renders incrementally.
When add agent tools?
When users need actions beyond Q&A. HITL on writes. Cap tool calls. Fallback to RAG-only if tools fail.
Memory architecture?
Short: session buffer in Redis. Long: vector memory for prefs. Summarize when >8K tokens.
Eval strategy?
Golden 500 Q&A. Faithfulness + Recall@5 CI gate. Online thumbs feedback. Weekly human spot check.
Cost optimization?
Semantic cache, query router, smaller model for easy queries, rerank top-20, batch embed ingest.
SDE3: 12-month evolution?
M1–3: RAG baseline + eval. M4–6: cache + router + memory. M7–9: agent tools + HITL. M10–12: fine-tuned reranker, multi-region, 50% cost reduction.
