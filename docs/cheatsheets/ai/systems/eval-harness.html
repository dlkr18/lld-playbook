Eval Harness & Golden Sets // Systems

Golden datasets, automated metrics, regression gates, and human eval loops. You cannot improve what you do not measure — especially for non-deterministic LLM outputs.

Golden SetRagasRegressionLLM JudgeCI Gate

Eval harness: golden Q&A dataset, retrieval metrics, generation metrics (faithfulness, relevance), CI regression gate, human spot-check queue, production feedback loop.

01 — What & Why

LLM outputs are stochastic. Eval harness provides reproducible quality measurement across prompt, model, and retrieval changes.

03 — Architecture

Golden Set -> Eval Runner -> [Retrieval metrics | Generation metrics]
              -> Report -> CI Gate (pass/fail) -> Human review queue

04 — Deep Dive: Golden Sets

Golden set structure: question, gold_answer, gold_chunk_ids, difficulty tag, tenant

200–500 pairs minimum. Tags: easy/medium/hard, single-hop/multi-hop. Synthetic bootstrap; human label source of truth.

05 — Deep Dive: Retrieval Metrics

Recall@k, MRR, nDCG@k. Requires gold_chunk_ids per question. Run on every index/chunk change.

06 — Deep Dive: Generation Metrics

Faithfulness, answer relevance, context precision/recall. Ragas, DeepEval, Phoenix.

07 — Deep Dive: LLM-as-Judge

GPT-4 judge scores faithfulness 1-5 with rubric and chain-of-thought reasoning

Use stronger model as judge. Rubric in prompt. Correlate judge with human labels monthly.

08 — Deep Dive: CI Regression Gate

Block merge if Recall@5 drops >2% or faithfulness drops >5%. Run on PR with smaller smoke set (50 cases).

09 — Production & Ops

  • Weekly human spot-check 50 production queries
  • Thumbs down → auto-add to review queue
  • Eval dashboard per model version

11 — Common Pitfalls

  • Eval leakage — golden Q derived from dev chunks
  • Single metric optimization
  • Flaky LLM judge without calibration
  • No eval before prompt change

12 — Interview Q&A

What belongs in a RAG golden set?
question, gold_answer, gold_chunk_ids, optional difficulty/tags. 200–500 human-labeled pairs held out from tuning.
Key metrics for RAG?
Retrieval: Recall@5, nDCG@5. Generation: faithfulness, answer relevance, context precision. Ops: p99 latency, cost/query.
LLM-as-judge reliability?
Calibrate against human labels monthly. Use rubric + stronger judge model. Ensemble 3 judges for high-stakes.
When block a deploy?
Faithfulness drop >5%, Recall@5 drop >2%, safety failure on red-team set, latency p99 regression >20%.
Synthetic eval bootstrap?
LLM generates Q from docs; human verifies. Fast bootstrap but replace with human gold over time.
Eval agents vs RAG?
Agents: task success rate, tool call accuracy, steps to completion, cost. Trajectory match against golden paths.
Ragas vs DeepEval?
Both solid. Ragas RAG-focused metrics. DeepEval broader test cases. Pick one; integrate in CI.
How avoid eval leakage?
Hold-out golden set never used in prompt tuning. Separate synthetic generation docs from eval docs.
Human eval cadence?
Weekly 50-sample spot check. Full re-label quarterly. Inter-annotator agreement tracking.
Online vs offline eval?
Offline golden set pre-deploy. Online A/B thumbs + implicit signals post-deploy. Both required.
Eval cost control?
Smoke set 50 cases on every PR. Full 500-case nightly. Sample production 1% for continuous eval.
SDE3: build eval platform?
Unified runner API, versioned golden sets, metric plugins, CI gate integration, human review UI, production feedback ingestion, per-team dashboards.
