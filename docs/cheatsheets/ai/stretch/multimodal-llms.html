Multimodal LLMs // Stretch

Models that reason jointly over text, images, audio, and video: vision encoders fused into decoder-only LLMs (GPT-4o, Claude 3.5, Gemini), API patterns for base64/URL media, and production trade-offs vs separate specialist pipelines.

Vision GPT-4o Audio Fusion

Unified multimodal stack: modality-specific encoders (ViT for images, Whisper-style for audio) project into the LLM’s token embedding space; the decoder attends across text + visual/audio tokens autoregressively. API layer accepts mixed content parts in one messages array.

01 — What & Why

Problem: Real-world tasks need more than text — receipts, UI screenshots, medical imaging, voice notes. Multimodal LLMs let one model describe, extract, and reason without brittle OCR + rules pipelines.

Functional

  • Answer questions about images (charts, diagrams, product photos)
  • Extract structured data from documents (invoices, forms)
  • Transcribe + summarize audio in one pass (native or chained)
  • Compare multiple images or frames in context

Non-Functional

  • Latency: image prefill is heavy — TTFT 1–4s for high-res
  • Cost: vision billed per image tile/patch or fixed image tokens (e.g. ~765 tokens/image)
  • Accuracy: OCR and fine detail still fail on small text; zoom/crop preprocessing helps
  • Privacy: PII in images — redact before upload; on-prem models for regulated domains
Interview anchor: Multimodal is not magic OCR — it is shared representation learning where visual patches become tokens the same decoder uses for language.

04 — Deep Dive: Vision Encoders

Most frontier models use ViT-style patch encoders (14×14 or 16×16 patches). CLIP pre-training aligns image/text spaces before LLM fusion. Smaller patches → more tokens → better fine detail.

ModelVision approachNotes
GPT-4oNative multimodal; tile high-res imagesdetail: high increases patch budget
Claude 3.5+Strong document/chart understandingMultiple images per turn
Gemini 1.5+Long context + video framesUp to hours of video at low FPS
LLaVA / openViT + projector + LlamaSelf-host for data residency

Vision zoom: image split into patches → ViT layers → projected embeddings occupy consecutive token slots before user text. High-res tiling adds multiple grid sections; each section consumes additional image tokens.

06 — Deep Dive: Audio & Speech

Two patterns: native end-to-end (GPT-4o realtime audio) vs cascade (Whisper → LLM → TTS).

# Cascade (production default for cost control)
import openai

transcript = openai.audio.transcriptions.create(
    model="whisper-1", file=audio_file
).text

answer = openai.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role":"user","content": transcript + "\nSummarize in 3 bullets."}]
).choices[0].message.content
  • Whisper: ~$0.006/min; robust accents; word-level timestamps
  • Native audio: lower latency conversational UX; higher $ and less control

Audio zoom: waveform → mel spectrogram → encoder → text tokens OR direct speech-to-speech latent stream. Cascade adds 2–3 hops but easier to log, redact, and swap STT vendor.

07 — Deep Dive: Video & Documents

  • Video: sample 1 fps (or scene-change keyframes); each frame = image tokens — explodes context
  • PDF: rasterize pages or use native PDF support; batch 5–10 pages per call
  • Preprocessing: crop to ROI, enhance contrast for OCR-like tasks
  • RAG hybrid: embed image captions from vision model into vector index for search; LLM for final QA

08 — Production & Ops

Cost control

  • Default detail: low; escalate to high only when OCR fails
  • Resize client-side to max useful dimension (e.g. 2048px long edge)
  • Cache vision encodings if same image re-queried (provider-dependent)

Safety

  • Visual jailbreaks (text in image bypasses filters) — run moderation on extracted intent
  • CSAM / biometric: block uploads; use provider safety APIs

Eval

  • Golden set of (image, question, expected JSON) pairs
  • Track field-level accuracy on structured extraction, not BLEU

09 — Where It Shows Up

  • LLM Fundamentals — decoder-only base; multimodal extends input tokens
  • Embeddings & Semantic Search — CLIP-style dual encoders for image search
  • RAG End-to-End — chart/table QA with retrieved pages as images
  • Instagram Feed HLD — vision embeddings for content understanding at scale
  • YouTube/Netflix HLD — thumbnail/frame understanding vs full multimodal chat

10 — Common Pitfalls

  • Trusting numeric OCR: always validate totals with code when money involved
  • Full-resolution uploads: 12MP photo → unnecessary tokens; resize first
  • Single-frame video: misses temporal context — sample multiple frames
  • No structured output schema: use JSON mode + validate with Pydantic
  • Ignoring EXIF/PII: strip metadata before cloud upload

11 — Interview Q&A

How do vision tokens enter a decoder-only LLM?
A vision encoder converts image patches into vectors; a projection layer maps them to the LLM’s embedding dimension. These vectors occupy sequential positions in the input (like text tokens). The causal decoder attends to all prior positions during prefill, then generates text autoregressively.
GPT-4o vs Whisper + GPT-4o-mini for voice apps?
Native audio: best for low-latency conversational UX, single vendor stack. Cascade: Whisper STT + text LLM + TTS — cheaper at scale, easier to log/redact transcripts, swap components, and debug. Most enterprise apps start cascade; move native when latency UX demands it.
What drives multimodal API cost?
Image token count (resolution, tiling, detail flag), number of frames for video, plus output tokens. A high-detail page can cost 2–5× low detail. Preprocess images client-side; batch extractions into one call when possible.
When would you NOT use a multimodal LLM for documents?
High-volume deterministic extraction (millions of forms) — use specialized OCR + templates with LLM fallback. Strict regulatory audit trails may require on-prem models. When field-level 99.9% accuracy is required without human review.
Explain early fusion vs cross-attention fusion.
Early fusion: visual and text tokens share one transformer stack from the start (GPT-4o style). Cross-attention: frozen image encoder; text layers attend to image features (Flamingo). Early fusion simplifies serving; cross-attention was popular for efficient adaptation of frozen LLMs.
How do you evaluate vision LLM quality in production?
Curate labeled set with field-level ground truth (not just human preference). Metrics: exact match on JSON fields, numeric tolerance for amounts, recall on listed items. A/B test preprocessing (crop, contrast). Monitor regression when provider updates model snapshot.
How does CLIP relate to multimodal chat models?
CLIP aligns image and text in a shared embedding space for retrieval/zero-shot classification. Chat multimodal models go further: they fuse visual tokens into a generative decoder for reasoning and dialogue. CLIP-style pre-training is often a prequel stage, not the final product.
What is the context window impact of 10 images?
Each image consumes hundreds to low-thousands of tokens depending on resolution. Ten high-detail screenshots can exceed 20K tokens before user text — crowding RAG context. Strategy: selective images, low detail first, summarize images into text for follow-up turns.
Design a receipt processing pipeline with multimodal LLMs.
Upload → resize → virus scan → GPT-4o with JSON schema extraction → Pydantic validate → rules engine for tax/totals → human queue if confidence low. Cache by image hash. Do NOT trust LLM arithmetic; recompute totals from line items in code.
Visual prompt injection — mitigations?
Adversarial text rendered in images (“ignore instructions”). Mitigate: OCR pre-pass with separate moderation, don’t let image text override system policy, use structured tasks (extract field X only), log and block known attack patterns.
Multimodal RAG: how to index images?
Generate captions/tags with vision LLM offline; embed captions with text embedder; store image URI + caption in vector DB. Query: text → retrieve captions → pass corresponding images to vision LLM for final answer. See RAG End-to-End and Vector Databases.
SDE3: multimodal vs specialist pipeline trade-off?
Multimodal LLM wins on flexibility and time-to-market for varied visual QA. Specialist stack (OCR + NER + rules) wins on cost, latency, and deterministic accuracy at high volume. Hybrid: specialists for hot path, multimodal for edge cases and new doc types. Always separate perception from business logic validation.
