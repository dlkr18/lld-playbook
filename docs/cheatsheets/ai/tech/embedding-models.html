Embedding Models // AI Tech

Choose and deploy text embedding models: OpenAI text-embedding-3, Cohere embed-v3, open-source BGE/E5, Matryoshka dimensions, cosine metrics, and eval before you pick an index.

text-embedding-3BGECosineMatryoshkaMTEB

Embedding model selection: text → tokenizer → encoder → L2-normalized vector (384–3072 dim) → cosine similarity in vector DB. Match model at ingest and query; version in index namespace.

01 — What & Why

Retrieval quality ceiling is set by embedding model before chunking tuning or rerankers. Wrong model or metric wastes index work. Primer: Embeddings & Semantic Search.

12 — Interview Q&A

How pick embedding model?
Start text-embedding-3-small or bge-small; eval recall@5 on golden set; upgrade if < target; consider cost and self-host constraints.
OpenAI small vs large?
Large ~+3–5% recall on hard sets, 2× cost and storage. A/B on your data before defaulting to large.
What is Matryoshka?
Embeddings trained so first k dimensions still useful — truncate 1536→512 for cheaper index with small recall trade.
Cosine vs dot product?
For normalized vectors, equivalent. Use index ops matching your normalization pipeline.
Cohere input_type?
search_document at ingest, search_query at query time — asymmetric embed improves retrieval.
Open source when?
Air-gap, cost at billions of tokens, fine-tune on domain. Pay GPU ops.
Fine-tune embeddings?
Rare; only with large labeled query-doc pairs. LoRA on encoder; re-eval vs bigger off-shelf.
Multilingual?
embed-multilingual-v3 or multilingual-e5-large; eval per language slice.
Embed long documents?
Chunk first — embed models have 512–8192 token limits. Never embed 100-page PDF whole.
Cache embeddings?
Cache query vectors by hash(question); invalidate on model change.
MTEB trust?
Starting point only — always domain golden set. Legal/medical need specialty eval.
SDE3: embed platform for company?
Central embed API; model registry; batch GPU workers; versioned namespaces; automatic re-index job on model bump; cost allocation per team.
