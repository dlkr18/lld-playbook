Chroma // AI Tech

Embedded open-source vector database for local dev and prototypes: pip install, in-process or client/server mode, collections, metadata filters, and LangChain/LlamaIndex integration.

EmbeddedCollectionsDevLangChainLocal

Chroma: Python client → Collection (embed + metadata + ids) → HNSW in-process or Chroma server → query top-k. Ideal for notebooks and CI; graduate before production SLAs.

01 — What & Why

Chroma is the fastest path to a working vector store in Python. Zero infra for demos and tests. Not a substitute for Pinecone or pgvector at multi-tenant production scale.

01b — API Surface

import chromadb
from chromadb.utils import embedding_functions

ef = embedding_functions.OpenAIEmbeddingFunction(
    api_key="...", model_name="text-embedding-3-small")
client = chromadb.PersistentClient(path="./chroma_db")
col = client.get_or_create_collection("docs", embedding_function=ef)

col.add(ids=["1"], documents=["Refund within 30 days"], metadatas=[{"type": "policy"}])
results = col.query(query_texts=["return policy"], n_results=5)

05 — Embedding Functions

Embedding function pipeline: document text → OpenAI embed → store vector on add and query

Bring your own vectors with add(embeddings=...) for prod model parity testing.

06 — Client/Server

HttpClient to chroma run --host — shared dev server. Not HA production cluster.

07 — Integrations

LangChain Chroma vectorstore; LlamaIndex ChromaVectorStore. Swap to Pinecone with same chunk pipeline.

08 — Graduation Path

Prototype Chroma → swap VectorStore backend to pgvector/Pinecone; keep same chunk IDs and metadata schema.

09 — Production & Ops

Chroma not recommended for prod SLAs. If forced: persistent volume, single-tenant, backup collection dir.

12 — Interview Q&A

What is Chroma best for?
Local dev, notebooks, CI tests, hackathons — vector search in 5 lines without cloud signup.
Persistent vs in-memory?
PersistentClient saves to disk; EphemeralClient for tests. In-memory lost on exit.
How filter metadata?
query(where={"field": "value"}) or where_document for text contains.
Chroma vs FAISS?
Chroma: metadata, persistence, API. FAISS: raw index library — faster but DIY everything.
Production ready?
Generally no for multi-tenant SaaS — no managed HA, limited scale. Use pgvector/Pinecone.
Embedding function gotcha?
Chroma can embed for you — ensure same model as production pipeline.
LangChain integration?
from langchain_chroma import Chroma — drop-in with retriever pattern.
Migrate to Pinecone?
Export ids, embeddings, metadata; upsert to Pinecone namespace; switch retriever config.
Client/server mode?
Docker chroma for team; still not production cluster replacement.
Collection sizing?
Fine to ~500K vectors local; slows beyond — signals graduation time.
Backup?
Copy persist directory; no built-in replication.
SDE3: when allow Chroma?
Only dev/staging with synthetic data; contract test that retriever interface swaps to pgvector in prod config.
