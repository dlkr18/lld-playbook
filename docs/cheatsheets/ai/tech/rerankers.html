Rerankers // AI Tech

Two-stage retrieval: bi-encoder pulls top-50 fast, cross-encoder reranker scores query-passage pairs precisely → top-5 to LLM. The standard precision boost after vector DB.

Cross-EncoderTwo-StageCoherebge-rerankerLatency

Two-stage pipeline: vector DB top_k=50 (~20ms) → cross-encoder rerank (~50–80ms for 50 pairs) → top 5 chunks to LLM. Typical +15–25% nDCG vs bi-encoder alone.

01 — What & Why

Bi-encoders embed query and doc separately — fast but shallow similarity. Cross-encoders joint-attend query+passage — slow but accurate. Always rerank after ANN in production RAG when latency budget allows ~50–100 ms.

03 — Architecture

Query -> Embed -> Vector DB (k=50) -> Reranker (n=5) -> Prompt -> LLM

04 — Deep Dive: Cross-Encoder

Cross-encoder joint attention on query+passage pair; scores relevance 0-1

Cannot pre-compute doc side — must run per query-passage pair at query time.

05 — Models

Pipeline: bi-encoder k=50, Cohere rerank top_n=5, latency budget 120ms total retrieve stage

Cohere rerank-english-v3.0, bge-reranker-v2-m3, ms-marco-MiniLM — pick by language and latency.

06 — Pipeline Tuning

Sweep retrieve_k ∈ {20,50,100} and rerank_n ∈ {3,5,8} on golden nDCG. Diminishing returns above k=50 for most corpora.

07 — Latency

50 passages × cross-encoder ~50–80 ms GPU; batch pairs. Cohere API ~100–200 ms. Budget total retrieve+rerank <150 ms p99.

08 — Hosted vs Local

Cohere: zero ops. Local GPU: lower $ at high QPS. CPU MiniLM for dev only.

09 — Production & Ops

  • Log rerank scores for drift detection
  • Fallback to ANN order if reranker timeout
  • A/B rerank on/off for quality metrics

12 — Interview Q&A

Bi-encoder vs cross-encoder?
Bi-encoder: embed separately, fast ANN. Cross-encoder: joint model on pair, accurate slow. Retrieve with bi-encoder, rerank with cross-encoder.
Why not only cross-encoder?
Can't pre-index passages — linear scan over corpus too slow. Two-stage is industry standard.
How many candidates to rerank?
Start 50 → rerank to 5. Increase k if recall@50 low on eval; decrease if latency high.
Cohere vs bge-reranker?
Cohere hosted easy. bge self-host on GPU for cost at scale and air-gap.
Latency budget?
Target retrieve+rerank <150 ms p99. Batch cross-encoder pairs; GPU inference server.
Rerank after hybrid?
Yes — hybrid improves recall; rerank improves precision on fused list.
Score threshold?
Drop passages below score 0.3 — reduces hallucination from irrelevant chunks.
Multilingual rerank?
bge-reranker-v2-m3 or Cohere multilingual — match query language.
Fine-tune reranker?
With click/logs pairs — smaller lift than fixing chunking/embed; advanced optimization.
Failure fallback?
On reranker timeout return ANN top-5 — degrade gracefully; alert.
Eval metric?
nDCG@5, MRR on golden set comparing with/without rerank.
SDE3: rerank service?
Stateless GPU pods; gRPC batch rerank; cache not useful (query-specific); SLO 80ms p99; autoscale on QPS; shadow test new model versions.
