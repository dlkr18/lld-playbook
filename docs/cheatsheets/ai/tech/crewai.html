CrewAI // AI Tech

Role-based multi-agent framework: Agents with backstory, Tasks with expected output, Crew orchestration (sequential, hierarchical). Fast path from idea to research-style agent teams.

AgentsTasksCrewHierarchicalDelegation

CrewAI flow: define Agents (role, goal, tools) → Tasks (description, agent, context) → Crew(process=sequential|hierarchical) → kickoff() → consolidated output.

01 — What & Why

CrewAI models teams of LLM agents with roles (Researcher, Writer, Manager). Tasks chain outputs; hierarchical process adds a manager agent that delegates. Great for prototypes; production often migrates to LangGraph for HITL and checkpointing.

01b — API Surface

from crewai import Agent, Task, Crew, Process

researcher = Agent(role="Researcher", goal="Find facts", backstory="...", tools=[search])
writer = Agent(role="Writer", goal="Draft report", backstory="...")

t1 = Task(description="Research {topic}", expected_output="Bullet facts", agent=researcher)
t2 = Task(description="Write report", expected_output="Markdown report", agent=writer, context=[t1])

crew = Crew(agents=[researcher, writer], tasks=[t1, t2], process=Process.sequential)
result = crew.kickoff(inputs={"topic": "RAG trends"})

04 — Deep Dive: Agents

Agent cards: role, goal, backstory, tools, allow_delegation flag

Each agent gets subset of tools. allow_delegation=True lets agent ask another agent (use sparingly — cost).

05 — Deep Dive: Tasks

expected_output acts as implicit rubric. context=[task1] passes prior output. Async execution via async_execution=True on independent tasks.

06 — Deep Dive: Process Types

Sequential vs hierarchical: manager agent routes tasks to researcher and writer

Sequential: fixed order. Hierarchical: manager_llm assigns dynamically. Cap total tasks at 5–8 for cost control.

07 — Deep Dive: Tools

CrewAI tools wrap LangChain tools or custom BaseTool. Same security rules: validate args, read-only default, HITL on writes.

12 — Interview Q&A

What is CrewAI optimized for?
Role-based multi-step workflows: research → analyze → write. Declarative agents/tasks faster than hand-rolling LangGraph for demos.
Explain Agent vs Task.
Agent = persona + capabilities (tools). Task = unit of work with description and expected_output assigned to one agent. Tasks can depend on prior task outputs via context=[].
Sequential vs hierarchical process?
Sequential runs tasks in list order. Hierarchical adds manager agent that delegates tasks to workers dynamically — more flexible, less predictable.
How control cost?
Limit agents/tools, cap kickoff timeout, use cheaper model for manager, disable allow_delegation unless needed, cache research task outputs.
CrewAI vs AutoGen?
CrewAI: structured roles/tasks, simpler API. AutoGen: conversational agents, group chat patterns. Both less production-control than LangGraph.
Can CrewAI use local models?
Yes via LangChain LLM wrappers (Ollama, vLLM OpenAI-compatible endpoint). See Model Serving.
How test crew output quality?
Golden inputs with rubric on expected_output; LLM-as-judge; human review sample; regression when prompt/backstory changes.
Tool security?
Same as any agent: least privilege, validate JSON args, no arbitrary code exec without sandbox.
When migrate to LangGraph?
Need HITL approval, durable checkpoint across days, custom routing logic, strict audit of each step.
Memory across kickoffs?
Pass prior result in inputs or external DB; CrewAI doesn't replace session store for user chat.
Parallel tasks?
async_execution=True on tasks without context dependency — runs in parallel where safe.
SDE3: design content pipeline with CrewAI.
Researcher (web+RAG) → Fact-checker → Writer → Editor tasks; sequential; hierarchical manager for retry on failed fact-check; output to CMS API with HITL gate before publish (LangGraph for gate if Crew insufficient).
