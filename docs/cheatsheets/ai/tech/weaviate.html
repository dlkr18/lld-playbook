Weaviate // AI Tech

Open-source vector database with native hybrid search (BM25 + vector), GraphQL API, modular embedders, and multi-tenancy. Self-host or Weaviate Cloud.

HybridGraphQLBM25Multi-tenantHNSW

Weaviate: GraphQL/REST API → hybrid query (alpha blends BM25 + vector) → HNSW index → filtered objects with properties. Self-host or managed cloud.

01 — What & Why

Weaviate shines when you need hybrid retrieval in one query, GraphQL flexibility, and optional server-side vectorization modules. Heavier ops than Pinecone if self-hosted.

03 — Architecture

Go binary cluster; replication; HNSW + inverted index for BM25. Modules call OpenAI/Cohere for vectorization at ingest.

04 — Deep Dive: Hybrid Search

Hybrid alpha slider: 0=BM25 only, 1=vector only, 0.5=fusion default starting point

Start alpha=0.5; grid search {0.3,0.5,0.7} on golden Recall@5.

05 — Deep Dive: GraphQL API

GraphQL Get query with where filter, hybrid block, and _additional score metadata

Also REST v1 objects API. GraphQL good for flexible property selection.

06 — Multi-Tenancy

Enable multiTenancyConfig on class; each tenant isolated. Auto-tenant creation on ingest optional.

07 — Modules

text2vec-openai, reranker-cohere, generative-openai. Can vectorize at ingest server-side — still version embed model carefully.

08 — vs Others

vs Pinecone: more hybrid-native, more ops. vs pgvector: no SQL joins but better hybrid DX.

12 — Interview Q&A

Weaviate's killer feature?
Native hybrid search in one API call — BM25 + vector with alpha tuning. Avoids dual-system fusion code.
What is alpha in hybrid?
Weight between dense (1) and sparse (0). 0.5 default; SKU queries lower alpha (more BM25).
Self-host vs cloud?
Self-host: K8s, replication, you patch. Cloud: managed backups, scaling — like Pinecone model.
GraphQL vs REST?
GraphQL flexible for nested queries and property pick. REST simpler for CRUD scripts.
Multi-tenancy model?
Tenant shards per class; query must specify tenant. Strong isolation for SaaS.
Server-side vectorization?
Modules embed at ingest — ensure query uses same module/model or bring own vectors.
Weaviate vs Pinecone?
Weaviate: hybrid-native, self-host option. Pinecone: simpler managed, less ops.
Weaviate vs pgvector?
Weaviate: hybrid + GraphQL. pgvector: SQL joins + ACID in Postgres.
Reranking integration?
reranker-cohere module or external cross-encoder after hybrid top-20.
Index rebuild?
Re-import with new vectorizer; new class version; blue/green cutover.
Ops monitoring?
Prometheus metrics; query latency; object count; hybrid alpha A/B in logs.
SDE3: hybrid RAG on Weaviate?
Class Document with text property + tenantId; multiTenancy; hybrid alpha tuned per query router; retrieve 20 → Cohere rerank → 5 to LLM; eval Recall@5 weekly.
