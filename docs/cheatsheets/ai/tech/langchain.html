LangChain // AI Tech

Composable LLM framework: runnables (LCEL pipe), prompts, retrievers, tools, memory, and callbacks. The component layer most orchestrators (LangGraph, LlamaIndex integrations) build on.

LCELRunnableRetrieversToolsCallbacks

LangChain stack: Prompt | ChatModel | OutputParser as LCEL chain; retriever feeds context; tools bound via bind_tools; callbacks → LangSmith traces.

01 — What & Why

LangChain standardizes LLM app primitives: models, prompts, parsers, retrievers, tools — as composable Runnable objects. Use LCEL (prompt | model | parser) for DAG pipelines; use LangGraph when you need cycles and checkpointing.

01b — API Surface

from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

prompt = ChatPromptTemplate.from_messages([
    ("system", "Answer from context: {context}"),
    ("human", "{question}"),
])
chain = prompt | ChatOpenAI(model="gpt-4o-mini") | StrOutputParser()
chain.invoke({"context": docs, "question": q})

07 — Deep Dive: Memory

ChatMessageHistory, ConversationBufferWindowMemory (legacy). Production: store messages in Postgres; pass history in invoke input — don't rely on in-process memory.

08 — Deep Dive: Callbacks

LangSmith auto-traces when env vars set. Custom BaseCallbackHandler for token logging. Tags: chain.invoke(..., config={"tags": ["prod"]}).

12 — Interview Q&A

What is LCEL?
LangChain Expression Language: compose Runnables with | operator. prompt | model | parser. Supports sync/async/stream/batch uniformly.
LangChain vs LangGraph?
LangChain = components and DAG chains. LangGraph = stateful graph with cycles, checkpointing, HITL. Use both: LC inside LangGraph nodes.
How wire a vector DB retriever?
VectorStore.from_documents + as_retriever(search_kwargs={"k": 20}). In chain: {"context": retriever | format_docs, "question": RunnablePassthrough()}.
What are callbacks for?
Hooks on chain start/end, LLM start/end, tool calls. LangSmith uses them for tracing; custom handlers for metrics.
Is LangChain still relevant in 2025?
Yes as a component library; less as opaque agents. Teams use lc-core + langgraph + direct API where simpler.
How handle streaming in LCEL?
chain.stream(input) yields partial dicts. For token stream: model streaming step or astream_events API.
Retriever vs VectorStore?
VectorStore = storage + similarity_search. Retriever = Runnable interface for chains (includes MMR, compression, multi-query wrappers).
Tool binding best practice?
Clear descriptions, minimal parameters, validate args before side effects. bind_tools on model; execute in controlled ToolNode.
Memory in production?
External store (Redis/Postgres) keyed by session_id; pass messages in state — not ConversationChain in-memory.
Version churn risk?
Pin langchain-core, langchain-openai; integration tests on golden prompts; avoid deprecated chains in docs.
LangChain vs LlamaIndex?
LangChain general orchestration + tools. LlamaIndex indexing/query focus. Overlap on RAG; many teams pick one primary.
SDE3: standardize LC across 10 teams?
Internal template repo: LCEL RAG chain, LangGraph agent, shared retriever factory, LangSmith project per env, banned patterns list (deprecated agents).
